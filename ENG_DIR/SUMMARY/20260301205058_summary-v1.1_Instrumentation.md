## Feature 2 — Instrumentation (Page View Tracking + Popularity Ranking)

### Database Migration
**File:** `supabase/migrations/20260301000000_instrumentation_tags_notifications.sql`

```sql
CREATE TABLE public.page_view (
  id           uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  entity_type  text        NOT NULL CHECK (entity_type IN ('subxeuron', 'publication', 'event', 'post')),
  entity_id    uuid        NOT NULL,
  viewer_id    uuid        REFERENCES public."user"(id) ON DELETE SET NULL,
  session_id   text,
  created_at   timestamptz NOT NULL DEFAULT now()
);
```
- RLS enabled: anonymous inserts allowed; reads restricted to service role
- Indexed on `(entity_type, entity_id)` and `created_at DESC`

### New Files
| File | Purpose |
|------|---------|
| `app/api/track/route.ts` | `POST /api/track` — records a page view for any entity |
| `components/shared/TrackView.tsx` | Client component, fire-and-forget — drop on any page to track views |
| `lib/supabase/analytics.ts` | Analytics service with `getPopularEntityIds()`, `getTotalViews()`, `getEntityViewBreakdown()` |

### Modified Files
| File | Change |
|------|--------|
| `lib/supabase/subxeurons.ts` | Added `getSubxeuronsByIds(ids[])` — fetches by ID list, preserving order |
| `lib/supabase/publications.ts` | Added `getPublicationsByIds(ids[])` — same pattern |
| `app/(app)/popular/page.tsx` | Full rewrite — period tabs (24h / 7d / 30d), ranked by real view counts, falls back to most-recent if no data |

### Usage: TrackView component
```tsx
// Drop in any server page to track views on mount:
import { TrackView } from '@/components/shared/TrackView'
<TrackView entityType="publication" entityId={pub._id} />
```

### Analytics API
```ts
import { getPopularEntityIds, getEntityViewBreakdown } from '@/lib/supabase/analytics'

// Top 20 most-viewed publications in last 7 days
const popular = await getPopularEntityIds('publication', '7d', 20)
// → [{ entityId: '...', viewCount: 42 }, ...]

// View breakdown for a single entity
const breakdown = await getEntityViewBreakdown('event', eventId)
// → { h24: 5, d7: 38, d30: 120 }
```

### Popular Page
- URL: `/popular?period=7d` (default), `/popular?period=24h`, `/popular?period=30d`
- Shows period selector tabs at top
- SubXeurons and Publications ranked by actual view counts in the selected window
- Falls back to `created_at DESC` if no views recorded yet

| `action/createAnswer.ts` | After answer insert, fire-and-forget email to question author (skips if answerer = author) |

### Required Environment Variables
```env
RESEND_API_KEY=re_xxxxxxxxxxxx
RESEND_FROM_EMAIL=Xeuron <noreply@xeuron.com>
NEXT_PUBLIC_BASE_URL=https://www.xeuron.com
```
> Get a free API key at https://resend.com
> `RESEND_API_KEY` is optional — if unset, emails are silently skipped (safe for local dev)

### Notification Behavior
- Emails are non-blocking (fire-and-forget with `.catch()`) — never slow down the user action
- Self-comments/answers do not trigger notifications
- Gracefully handles missing email addresses (skips silently)

## Feature 4 — Word Tags for Search

### Database Migration
Same file as Feature 2 migration adds:

```sql
CREATE TABLE public.subxeuron_tag (
  id            uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  subxeuron_id  uuid        NOT NULL REFERENCES public.subxeuron(id) ON DELETE CASCADE,
  tag           text        NOT NULL,
  created_at    timestamptz NOT NULL DEFAULT now(),
  CONSTRAINT subxeuron_tag_unique UNIQUE (subxeuron_id, tag)
);

CREATE TABLE public.event_tag (
  id         uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  event_id   uuid        NOT NULL REFERENCES public.event(id) ON DELETE CASCADE,
  tag        text        NOT NULL,
  created_at timestamptz NOT NULL DEFAULT now(),
  CONSTRAINT event_tag_unique UNIQUE (event_id, tag)
);
```
- RLS: public reads, owner-only writes (moderator for subxeuron, creator for event)
- `publication_tag` already existed — same pattern applied to subxeurons and events

### New Files
| File | Purpose |
|------|---------|
| `lib/supabase/tags.ts` | Full CRUD + search for `subxeuron_tag` and `event_tag` |

### Tag Service API
```ts
import { addSubxeuronTag, searchSubxeuronsByTag, addEventTag, searchEventsByTag } from '@/lib/supabase/tags'

// After creating a subxeuron, add tags:
await addSubxeuronTag(subxeuronId, 'machine-learning')
await addSubxeuronTag(subxeuronId, 'robotics')

// Tag-based search:
const results = await searchSubxeuronsByTag('robot')
// → [{ subxeuronId: '...', tags: ['robotics', 'robot-perception'] }, ...]
```

### Modified Files

| `app/(app)/search/page.tsx` | Three search panels: SubXeurons (FTS + tag merged), Publications (FTS), Events by tag |
