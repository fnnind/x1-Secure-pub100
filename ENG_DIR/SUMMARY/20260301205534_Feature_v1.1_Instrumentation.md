## Feature 2 — Instrumentation (Page View Tracking + Popularity Ranking)

### Database Migration
**File:** `supabase/migrations/20260301000000_instrumentation_tags_notifications.sql`

```sql
CREATE TABLE public.page_view (
  id           uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
  entity_type  text        NOT NULL CHECK (entity_type IN ('subxeuron', 'publication', 'event', 'post')),
  entity_id    uuid        NOT NULL,
  viewer_id    uuid        REFERENCES public."user"(id) ON DELETE SET NULL,
  session_id   text,
  created_at   timestamptz NOT NULL DEFAULT now()
);
```
- RLS: anonymous inserts allowed; reads restricted to service role
- Indexed on `(entity_type, entity_id)` and `created_at DESC`

### New Files
| File | Purpose |
|------|---------|
| `app/api/track/route.ts` | `POST /api/track` — records a page view for any entity |
| `components/shared/TrackView.tsx` | Client component, fire-and-forget — drop on any page to track views |
| `lib/supabase/analytics.ts` | `getPopularEntityIds()`, `getTotalViews()`, `getEntityViewBreakdown()` |

### Modified Files
| File | Change |
|------|--------|
| `lib/supabase/subxeurons.ts` | Added `getSubxeuronsByIds(ids[])` — fetches by ID list, preserving rank order |
| `lib/supabase/publications.ts` | Added `getPublicationsByIds(ids[])` — same pattern |
| `app/(app)/popular/page.tsx` | Full rewrite — period tabs (24h / 7d / 30d), ranked by real view counts, falls back to most-recent if no data |

### Usage
```tsx
// Drop on any detail page to record a view:
<TrackView entityType="publication" entityId={pub._id} />

// Query popularity:
const popular = await getPopularEntityIds('publication', '7d', 20)
// → [{ entityId: '...', viewCount: 42 }, ...]
```

### Popular Page
- URL: `/popular?period=7d` (default), `?period=24h`, `?period=30d`
- Period selector tabs shown above the listing
- Ranked by view counts; falls back to `created_at DESC` when no views recorded yet

### Templates
- `buildCommentNotificationEmail()` — notifies post author on new comment

- Both include: date/time, content snippet, CTA button linking to the unique URL

### Modified Files
| File | Change |
|------|--------|
| `action/createComment.ts` | Fire-and-forget email to post author after comment insert |
| `action/createAnswer.ts` | Fire-and-forget email to question author after answer insert |

### Required Environment Variables
```env
RESEND_API_KEY=re_xxxxxxxxxxxx          # https://resend.com (free tier available)
RESEND_FROM_EMAIL=Xeuron <noreply@xeuron.com>
NEXT_PUBLIC_BASE_URL=https://www.xeuron.com
```
> If `RESEND_API_KEY` is unset, emails are silently skipped — safe for local dev.

### Behavior
- Non-blocking: fire-and-forget, never delays the user action
- Self-notifications suppressed (commenter = author → no email)
- Missing email addresses handled gracefully (skips silently)

## Feature 4 — Word Tags for Search

### Database Migration (same file as above)
```sql
CREATE TABLE public.subxeuron_tag (
  subxeuron_id  uuid  NOT NULL REFERENCES public.subxeuron(id) ON DELETE CASCADE,
  tag           text  NOT NULL,
  CONSTRAINT subxeuron_tag_unique UNIQUE (subxeuron_id, tag)
);

CREATE TABLE public.event_tag (
  event_id  uuid  NOT NULL REFERENCES public.event(id) ON DELETE CASCADE,
  tag       text  NOT NULL,
  CONSTRAINT event_tag_unique UNIQUE (event_id, tag)
);
```
- `publication_tag` already existed — same pattern now applied to subxeurons and events
- RLS: public reads, owner-only writes (moderator for subxeuron, creator for event)

### New Files
| File | Purpose |
|------|---------|
| `lib/supabase/tags.ts` | CRUD + tag search for `subxeuron_tag` and `event_tag` |

### Modified Files
| File | Change |
|------|--------|
| `app/(app)/search/page.tsx` | Three panels: SubXeurons (FTS + tag merged), Publications (FTS), Events by tag (new) |

### Search Behavior
- SubXeuron results = FTS ∪ tag-matched (deduplicated)
- Publication results = FTS unchanged
- Events panel appears when query is active, driven by tag matching
- URL: `/search?query=robotics`
