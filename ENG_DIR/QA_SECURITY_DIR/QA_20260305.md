  ---
  QA Review — Xeuron SEC_m2_pub100 (2026-03-05)                                                                                    
                  
  Security Issues                                                                                                                  
                  
  [SEC-1] No server-side source URL validation in createSubxeuron
  lib/supabase/mutations.ts:9-67 — createPublication validates sourceUrl against ALLOWED_SOURCE_HOSTS (line 150–163), but
  createSubxeuron has no equivalent server-side check. The only guard is client-side in CreateSubXeuronButton.tsx, which can be
  trivially bypassed by calling the Server Action directly. A user can create a subxeuron with any source_url.

  [SEC-2] HTML injection in email templates
  lib/email.ts:116-119 — User-supplied content (commentSnippet, answerSnippet, questionSnippet) is injected directly into HTML
  strings in buildCommentNotificationEmail and buildAnswerNotificationEmail without escaping. A user can inject arbitrary HTML into
   notification emails, including malicious links.
  // Unsafe:
  <div class="snippet">${truncate(p.commentSnippet)}</div>

  ---
  Performance Issues

  [PERF-1] N+1 queries in getPostComments / getCommentReplies
  lib/supabase/comments.ts:22-35 — For each top-level comment, three queries fire sequentially: two count queries via
  getCommentVotes + one via getUserCommentVoteStatus. For each reply, same pattern repeats via getCommentReplies. On a post with 20
   comments and 5 replies each = 75 queries minimum.

  [PERF-2] N+1 queries per Post in list contexts
  components/post/Post.tsx:24-26 — The Post server component calls getPostVotes (2 queries), getUserPostVoteStatus (1 query), and
  getPostComments (N+1 as above) per post. It's used in feed lists: SubxeuronPage, PapersTab, and event discussions pages. Each
  list renders posts.map((post) => <Post .../>) making this multiplicative.

  ---
  Correctness Issues

  [BUG-1] revalidatePath missing from most Server Actions
  Per CLAUDE.md convention, Server Actions must call revalidatePath after mutations. Currently missing from: upvote.ts,
  downvote.ts, deletePost.ts, deleteComment.ts, createComment.ts, createPost.tsx, voteEventQA.ts, submitPollVote.ts,
  toggleEventFavorite.ts. This means Next.js caches may serve stale data after mutations.

  [BUG-2] censorPost tool skips DB write when isToBeReported = false
  tools/tools.ts:23-25 — Early return when !isToBeReported means if the AI provides a censored title/body but sets isToBeReported:
  false, the censored content is never saved. Only reported posts get their content updated.

  - components/header/CreateSubXeuronButton.tsx:148 — console.log("start to create subXeuron")
  - components/header/CreateSubXeuronButton.tsx:210 — console.log("SubXeuron created:", result)
  - components/post/CreatePost.tsx:24 — console.log(...)

  [MINOR-2] Incomplete Portable Text rendering in Post.tsx
  components/post/Post.tsx:88 — Only renders post.body[0].children[0].text (first span of first block). Multi-paragraph posts

  [MINOR-3] CreateSubXeuronButton source URL allowlist narrower than server's
  CreateSubXeuronButton.tsx:109 — Client-side allowlist is ["doi.org", "arxiv.org", "nature.com"] but the server has a richer list
  for publications. Combined with SEC-1, the subxeuron source URL has no consistent validation.
